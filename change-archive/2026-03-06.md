### 11. CI 测试修复

### 12. GitHub Actions secrets 缺失兼容

修改：

- `.github/workflows/bump-version.yml`
- `.github/workflows/update-tokens.yml`
- `.github/workflows/skill-drift.yml`

问题原因：

- 新仓库未配置 `APP_ID` 与 `APP_PRIVATE_KEY`
- `actions/create-github-app-token@v1` 直接要求这两个输入
- job 级 `if` 里不能直接使用 `secrets.*`，否则 workflow 解析阶段就会报 `Unrecognized named-value: 'secrets'`
- 导致 workflow 在进入真正逻辑前就失败

修复方案：

- 去掉 job 级对 `secrets.*` 的直接判断
- 在每个 job 内新增 `Check GitHub App secrets` step
- 先把 secrets 注入 step 级 `env`
- 再通过 `GITHUB_OUTPUT` 生成 `has_app_secrets` 输出
- 后续需要 GitHub App token 的 steps 仅在 `has_app_secrets == 'true'` 时执行
- 未配置 secrets 时输出 skip 信息，而不是直接失败

验证结果：

- 三个 workflow 已改为 step 级 secrets 检查
- 避免再次触发 `Unrecognized named-value: 'secrets'` 解析错误

### 关键实现细节

- 新增 `QQBotChannel`，采用 **Webhook 入站 + OpenAPI 出站**，并保持现有 channel 自注册模式。
- 入站支持 `GROUP_AT_MESSAGE_CREATE` 与 `C2C_MESSAGE_CREATE`；群聊消息会归一化为 `qqgrp:<group_openid>`，私聊消息归一化为 `qqdm:<user_openid>`。
- 保持 metadata-first 行为：先写入 chat metadata，仅对已注册 chat 投递 `onMessage`。
- 内置 `/chatid` 与 `/ping` 指令，延续 Telegram 风格的注册体验。
- `setup/verify.ts` 增加 `QQBOT_APP_ID` + `QQBOT_APP_SECRET` 检测，安装后验证能识别 `qqbot` 已配置。
- `.env.example` 增加 `QQBOT_APP_ID`、`QQBOT_APP_SECRET`、`QQBOT_WEBHOOK_HOST`、`QQBOT_WEBHOOK_PORT`、`QQBOT_WEBHOOK_PATH`。
- 为了让 webhook 验签在当前 Node 运行时中稳定可测，不再依赖 `crypto.generateKeyPairSync('ed25519', { seed })` 的隐式行为，改为显式使用 seed -> PKCS#8 DER -> `createPrivateKey()` / `createPublicKey()` 的方式派生 Ed25519 密钥对。
